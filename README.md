ВАРИАНТ 10: ТЕПЛИЦА
Проект представляет собой консольное приложение-симулятор управления теплицей.

ИСПОЛЬЗОВАННЫЙ СТЕК
Cats, Cats Effect, ZIO не используются. Работа построена исключительно на стандартной библиотеке Scala 3.

ИНФРАСТРУКТУРА
Реализованные монады:

Monad[M[_]] — Универсальный интерфейс для композиции вычислений. Заменяет сторонние библиотеки и задаёт единый контракт (pure, flatMap, map) для всех эффектов в проекте.

Монада Reader — протаскивание конфигурации
Reader[Env, A] — это обёртка вокруг функции Env => A. Монада позволяет связать множество проверок (нужен ли полив, достаточно ли света, готов ли урожай) в одну цепочку, не передавая настройки явно. Реальный объект конфигурации используется всего один раз в самом конце вычислений через метод .run(config).

Монада Writer — логирование и история действий
Контейнер Writer[A] всегда хранит пару: полезный результат A и накопленный лог Vector[String]. Когда два вычисления соединяются через .flatMap, монада автоматически склеивает логи первой и второй функции. Это позволяет формировать объяснимые отчёты о состоянии грядок без императивного println внутри бизнес-логики.

Монада State — память теплицы
State[S, A] — это обёртка над функцией вида S => (A, S). Она принимает текущее состояние теплицы (GreenhouseState), вычисляет новые параметры и возвращает кортеж: результат операции и обновлённое состояние. Метод .flatMap незаметно прокидывает это новое состояние на вход следующему шагу, полностью исключая использование var и глобальных переменных.

Монада IO — управление сайд-эффектами
Монада инкапсулирует внутри себя отложенное действие () => A. Программа не выполняет ввод-вывод сразу, а собирает порядок действий в единую цепочку. Вся императивная часть вынесена в отдельный слой и выполняется ровно один раз в методе @main через .unsafeRun().

ХРАНЕНИЕ ДАННЫХ
Для описания предметной области заведены ключевые структуры данных:

GreenhouseState — хранит актуальное состояние симуляции: список грядок (List[Bed]), текущий день, запасы воды и удобрений. Любое изменение создаёт новую копию состояния через .copy().

Config — содержит неизменяемые правила теплицы: нормы влаги по типам растений, минимальный уровень света, базовую скорость роста и порог зрелости для сбора урожая.

Cmd — структура команды меню. Позволяет описывать интерфейс декларативно через список данных, а не через императивные ветвления. Добавление нового действия требует лишь одной строки в списке команд.

ОСНОВНЫЕ ВОЗМОЖНОСТИ
- Уход за грядками: Полив (восстановление влаги до 100% с расходом запаса воды), внесение удобрений (ускорение роста с расходом запаса), включение света.
- Симуляция времени: Переход к следующему дню автоматически снижает влажность и освещённость, увеличивает рост растений и обновляет счётчик дней.
- Сбор урожая: Проверка зрелости через конфигурацию. Урожай собирается только при достижении порога роста, грядка безопасно удаляется из списка.
- Конфигурируемость: Все правила вынесены в Config. Добавление нового растения требует лишь одной строки в карте норм, без изменения кода логики.
- Чистая архитектура: Бизнес-логика полностью отделена от ввода/вывода. Отсутствие императивных конструкций (var, match-case, foldLeft). 
- Расширяемое меню: Интерфейс управляется списком List[Cmd]. Обработка ввода построена на Option.fold.
